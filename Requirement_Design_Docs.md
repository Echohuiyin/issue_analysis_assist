# Problem AI - Linux内核问题自动分析系统 需求设计文档

**版本**: V3.0  
**最后更新**: 2026-03-09

## 1. 项目概述

### 1.1 项目背景
Linux内核问题分析需要深厚的专业知识和丰富的经验积累，传统的人工分析方式效率低下且依赖个人能力。本项目旨在构建一个**Linux内核问题自动分析系统**，通过爬取社区优质案例，构建知识库，并结合LLM大模型能力，实现问题的自动分析、SKILL提炼和智能辅助。

### 1.2 项目目标
- 自动从技术社区（Stack Overflow、CSDN、知乎等）爬取Linux内核相关问题案例
- 构建结构化的案例知识库，存储问题信息、分析过程和解决方案
- 基于案例库生成RAG向量库，辅助问题分析
- 从案例中提炼Linux内核各模块的SKILL（存储、网络、调度、内存、锁、timer等）
- 结合RAG和SKILL，利用LLM实现智能问题分析

### 1.3 当前实现状态（V3.0）

**已完成功能**：
- ✅ 案例获取模块：支持StackOverflow、CSDN、知乎、掘金四个数据源
- ✅ 三表结构：RawCase（原始案例）、TrainingCase（训练数据）、TestCase（测试数据）
- ✅ 自动获取程序：fetch_raw_cases.py（支持智能延迟、去重、HTML解析）
- ✅ 自动处理程序：process_raw_cases.py（LLM解析、质量评估、训练/测试集划分）
- ✅ 向量嵌入：支持RAG检索

**下一步计划**：

```

│                          Linux内核问题自动分析系统                        │
├─────────────────┬─────────────────┬─────────────────┬─────────────────┤

         │                    │                    │                    │
```

## 2. 功能需求

### 2.1 案例获取模块（模块一）

#### 2.1.1 爬虫系统
负责从技术社区网站自动爬取Linux内核相关问题案例。

| 功能 | 描述 |
|------|------|
| 多源爬取 | 支持Stack Overflow、CSDN、知乎等平台 |
| 关键词匹配 | 根据预设关键词过滤Linux内核相关问题 |
| 内容解析 | 解析问题标题、描述、标签、答案等 |
| 自动去重 | 基于内容哈希去重，避免重复存储 |
| 增量爬取 | 支持增量更新，只获取新内容 |
| 爬取频率控制 | 遵守网站robots.txt，避免过度请求 |

#### 2.1.2 数据源配置
| 网站 | 爬取方式 | 数据字段 |
|------|----------|----------|
| Stack Overflow | API + 网页解析 | 标题、问题正文、答案、标签、投票数 |
| CSDN | 网页解析 | 标题、文章内容、评论、分类 |
| 知乎 | API + 网页解析 | 问题、回答、作者、标签 |
| 其他技术论坛 | 可扩展爬虫 | 自定义字段 |

#### 2.1.3 案例筛选
- **内容质量过滤**：过滤低质量、无答案、过于简单的问题
- **相关性过滤**：确保案例与Linux内核相关
- **时效性处理**：优先保留近期案例，定期清理过时内容

### 2.2 数据存储系统（模块二）

#### 2.2.0 三表结构设计（V3.0 已实现）

系统采用三表分离架构，实现原始数据、训练数据、测试数据的分离管理。

**三表架构**：
```
┌─────────────────┐
│   RawCase表     │  原始案例表
│   (原始数据)     │  - 存储从各数据源获取的原始内容
│                 │  - 状态跟踪：pending → processed
│                 │  - 去重机制
└────────┬────────┘
         │ LLM处理
         ▼
┌─────────────────┐
│  质量评估       │  质量分数 ≥ 70
│                 │  置信度 ≥ 0.7
└────┬───────┬────┘
     │       │
   80%      20%
     ▼       ▼
┌────────┐ ┌────────┐
│Training│ │  Test  │
│  Case  │ │  Case  │
│ 训练表  │ │ 测试表  │
└────────┘ └────────┘
```

**实现状态**：
- ✅ RawCase表：已实现，当前76条记录
- ✅ TrainingCase表：已实现，用于SKILL训练
- ✅ TestCase表：已实现，用于系统测试
- ✅ 自动获取程序：fetch_raw_cases.py
- ✅ 自动处理程序：process_raw_cases.py
- ✅ 质量评估机制：已实现
- ✅ 训练/测试集划分：已实现（80%/20%）

**关键需求**：
1. **原始案例表需求**：
   - 支持多数据源（StackOverflow、CSDN、知乎、掘金）
   - 存储原始HTML和纯文本内容
   - 状态管理（待处理、处理中、已处理、失败、低质量）
   - 内容去重（基于哈希）
   - 错误信息记录

2. **训练数据表需求**：
   - 结构化案例数据（标题、现象、日志、环境、根因、分析过程、解决方案）
   - 质量评分和置信度
   - 向量嵌入（用于RAG）
   - 内核模块分类
   - 严重程度分级

3. **测试数据表需求**：
   - 与训练数据表结构相同
   - 用于系统测试和验证
   - 独立于训练数据，避免数据泄露

#### 2.2.1 PostgreSQL案例存储（原设计）
负责结构化存储爬取的案例数据。

| 字段 | 类型 | 说明 |
|------|------|------|
| id | UUID | 主键 |
| source | VARCHAR(50) | 来源网站 |

#### 2.2.3 RAG向量库
基于PostgreSQL案例生成向量库，用于相似案例检索和输入扩展。

| 功能 | 描述 |
|------|------|
| 向量生成 | 使用Embedding模型将案例文本转为向量 |

| 相似检索 | 基于向量相似度检索相关案例 |
| Top-K返回 | 返回最相似的K个案例 |

### 2.3 SKILL提取模块（模块三）

#### 2.3.1 SKILL定义
SKILL是Linux内核问题分析的方法论和经验总结，包括：
- 问题分析方法
- 调试思路
- 常用工具
- 关键参数
- 常见根因

#### 2.3.2 SKILL结构
```json
{
  "module": "memory",
  "module_skill": {
    "analysis_method": ["vmstat分析", "slabtop使用", "页表分析"],
    "debug_tools": ["perf", "ftrace", "kmemleak", "vmstat", "top"],
    "key_parameters": ["vm.swappiness", "vm.dirty_ratio", "kmalloc"],
    "common_root_causes": ["内存泄漏", "页错误风暴", "OOM"],
    "solutions": ["释放缓存", "调整参数", "修复泄漏"]
  },
  "total_skill": "根据问题模块选择对应的module_skill进行分析..."
}
```

#### 2.3.3 SKILL提取流程
1. **读取案例**：从数据库读取已分类的案例
2. **提取分析**：分析案例的问题分析思路和过程
3. **提炼技能**：根据问题所属模块提炼该模块的SKILL
4. **生成总SKILL**：整合各模块SKILL生成总SKILL
5. **存储SKILL**：将SKILL结构化存储

#### 2.3.4 SKILL更新
- 支持增量更新：新案例入库后重新生成相关模块SKILL
- 支持批量更新：定期全量重新生成SKILL
- 版本管理：保留SKILL历史版本

### 2.4 问题分析模块（模块四）

#### 2.4.1 用户输入
| 输入类型 | 说明 |
|----------|------|
| 问题描述 | 用户描述的问题现象 |
| 日志信息 | dmesg、syslog、内核日志等 |
| 错误信息 | 错误代码、返回值 |
| 环境信息 | 内核版本、硬件配置 |

#### 2.4.2 分析流程
```
用户输入 → RAG扩展 → SKILL选择 → LLM分析 → 结果返回
    │          │          │          │
    ▼          ▼          ▼          ▼
 原始问题    相似案例    技能提示    分析报告
 扩展信息
```

#### 2.4.3 RAG增强
1. 将用户输入转为向量
2. 在向量库中检索相似案例
3. 将相似案例作为上下文提供给LLM

#### 2.4.4 SKILL辅助
1. 根据问题特征判断所属内核模块
2. 加载对应模块的SKILL
3. 将SKILL作为提示工程的一部分

#### 2.4.5 LLM分析
- 问题根因分析
- 影响范围评估
- 解决方案建议
- 相关案例推荐

#### 2.4.6 分析结果
| 输出项 | 说明 |
|--------|------|
| 问题分类 | 所属内核模块 |
| 根因分析 | 问题根本原因 |
| 影响评估 | 影响范围和严重程度 |
| 解决方案 | 建议的解决步骤 |
| 相似案例 | 相关的历史案例 |
| SKILL参考 | 使用的分析技能 |
