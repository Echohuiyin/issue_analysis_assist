## 问题诊断

### 原始问题
- 低质量案例数量：954个（占总数95.4%）
- 成功处理率：0.1%

### 根本原因
发现两个质量阈值设置不一致：
1. `cases/acquisition/validators.py` 中的阈值已降低到50
2. `process_raw_cases.py` 中的 `QUALITY_THRESHOLD` 仍为70

这导致即使验证器认为案例质量合格（≥50），处理器仍会因为阈值过高（≥70）而拒绝案例。

## 解决方案

### 修改内容
修改 `process_raw_cases.py` 第30行：
```python
# 修改前
QUALITY_THRESHOLD = 70.0  # 最低质量分数

# 修改后
QUALITY_THRESHOLD = 50.0  # 最低质量分数（从70降到50以提高通过率）
```

### 执行操作
重新运行 `reprocess_low_quality_cases.py`，将所有低质量案例状态改回pending并重新处理。
